# M6 — Polish & Ship Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Make the game shippable — a branded launcher, a Radix + framer-motion HUD (selection panel with build/research/rally controls, click-to-select), howler audio wired to every game event, SQLite + Preferences save/load, a credits screen — then deliver the debug Android APK and the GitHub Pages build in the single PR.

**Architecture:** The launcher is a phase before `playing` — `App` holds a `phase` state ('launcher' | 'playing'). Radix primitives (Dialog, Progress, Tooltip) style the HUD; framer-motion animates panel slide-ins and the launcher header. A howler audio module exposes four buses (sfx/music/ambient/ui); an event→sound map fires sounds from `runEconomyTick`'s damage events and game-state transitions. A `persistence` module wraps `@capacitor-community/sqlite` (save games) and `@capacitor/preferences` (settings) behind a platform-agnostic interface that falls back to the web implementations. Audio assets (OGG sfx + WAV music) are ingested into `public/assets/audio/`.

**Tech Stack:** Radix UI, framer-motion, howler, @capacitor-community/sqlite, @capacitor/preferences, the M0–M5 game.

**Source-of-truth:** `docs/specs/90-ui-hud.md`, `80-audio.md`, `95-persistence.md`, `docs/milestones/M6-polish.md`.
